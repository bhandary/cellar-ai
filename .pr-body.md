## Summary
- fix frontend SSE message handling so streamed assistant text attaches to the server-provided message id
- add focused frontend regression tests for assistant message creation, delta appends, and supported inline render blocks
- update CI to run the new frontend test suite before the production build

## QA finding addressed
During QA validation, the frontend created an optimistic assistant placeholder with a random local id before the SSE stream started. When `TEXT_MESSAGE_START` arrived, later text deltas switched to the backend `messageId`, but no matching message existed in state, so the streamed assistant response could remain blank in the chat UI even though the backend run completed successfully.
